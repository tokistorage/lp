3. TokiQR の動作要件

TokiQRは、ブラウザ上で音声の録音・符号化・QRコード生成を行うWebアプリケーションです。以下の要件があります。

録音機能

  • マイクへのアクセス許可（ブラウザの権限設定）
  • WebAssembly（WASM）対応ブラウザ — 上記の対応ブラウザすべてで利用可能

QRコードの読み取り

  • カメラ付きスマートフォン（2020年以降のモデルを推奨）
  • 標準カメラアプリまたはQRリーダーアプリ
  • 望遠レンズ搭載機種はより安定して読み取れます

オフライン対応

  • TokiQRはPWA（Progressive Web App）としてインストール可能です
  • 初回アクセス後はオフラインでも録音・再生が可能です
  • iOS：Safari → 共有ボタン → 「ホーム画面に追加」
  • Android：Chrome → メニュー（⋮） → 「ホーム画面に追加」

4. 印刷機能の推奨環境

QRコードやDIYキットの印刷には、以下の環境を推奨します。

  • 推奨：macOS + Google Chrome
  • 印刷レイアウトの再現性が最も高い組み合わせです
  • Safari では印刷時のレイアウトが意図通りに再現されない場合があります

5. 通信環境

  • 初回アクセス時にWASMファイル等のリソースをダウンロードします（合計約5MB）
  • 安定したインターネット接続を推奨します
  • 録音・符号化・QR生成はすべてブラウザ内で完結し、サーバーへの音声データ送信は行いません

6. JavaScript・Cookie

  • 本サイトの全機能を利用するには、JavaScriptを有効にしてください
  • お問い合わせフォームの入力内容記憶機能にCookieを使用しています（詳細はプライバシーポリシーをご覧ください）

7. 使用している主要技術

技術 用途
WebAssembly（WASM） Codec2音声符号化、Brotli圧縮、WebP画像変換
MediaRecorder API ブラウザ内音声録音
Web Audio API 音声再生・処理
Service Worker オフライン対応・リソースキャッシュ
GitHub Pages ホスティング

8. 既知の制限事項

  • 一部のアプリ内ブラウザ（LINE、Instagram等）では、マイクへのアクセスが制限される場合があります。その場合は、標準ブラウザ（SafariまたはChrome）で開いてください
  • iOS版Firefoxでは録音機能が制限される場合があります
  • 古いバージョンのブラウザではWebAssemblyが動作しない場合があります
