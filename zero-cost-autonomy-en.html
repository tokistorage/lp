Zero-Cost Autonomy

The Personal Infrastructure Built from Two Repositories — public and private
March 12, 2026  ·  Takuya Sato

I typed "good morning."

Thirty seconds later, ninety tasks were lined up in order of priority — along with a note that the tax filing deadline was three days away.

Not a single yen of additional cost was required to make this happen.

It started with a simple question: can Claude.ai run a git push?

The answer turned out to be asymmetric. Access to github.com went through. Access to api.github.com was restricted. That asymmetry determined the design. Claude operates via git. Anything requiring API access is delegated to GitHub Actions. Actions on public repositories run without limits — and without cost.

Two repositories were set up. tokistorage.github.io is public — the place where Actions run. tr is private — where data, skills, and memory live. Claude pushes a request to the public repo. Actions call api.github.com, write the result to the private repo. Claude pulls and reads.

Asynchronous — but that's enough.

The memory feature goes unused. Instead, context is written to memory/context/. Pull at the start of the next session, and the thread continues. The data lives in my own GitHub, not on Anthropic's servers. The format is plain text and JSON — always readable, always editable, with a full history in git log.

Skills live in the same place: how to write an essay, a brochure template, a guide for generating slide decks. Claude pulls them when needed. Capability accumulates in the repository — it doesn't disappear when the conversation ends.

A forty-year-old technology called git becomes the foundation for a personal AI agent.

"Turning Claude.ai into an agent" sounds ambitious. The reality is simple: a forty-year-old technology called git, the free Actions on a public repository, and the privacy of a private one. No new technology was used.

Infrastructure lasts longer when it's simple. When the cost is zero, there's no reason to stop.
